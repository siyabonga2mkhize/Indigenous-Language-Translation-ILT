### Issue 2: Design ER Diagram – Conceptual Model

| Field | Value |

| **Labels** | `database`, `documentation`, `enhancement` |
| **Milestone** | Phase 2 – System Design |
| **Priority** | High |
| **Description** | Create the conceptual ER diagram showing all entities, their attributes, and relationships (without implementation details). |

**Acceptance Criteria:**
- [ ] All entities from the planning document are represented
- [ ] All relationships are clearly shown with cardinality (1:1, 1:M, M:M)
- [ ] Primary keys are identified for each entity
- [ ] Foreign keys are identified for relationships
- [ ] Diagram is created using a tool (draw.io, Lucidchart, DBDiagram, etc.)
- [ ] Diagram is saved as `docs/diagrams/ER-Diagram-Conceptual.png`

**Entities to Include:**
```
User (ASP.NET Identity)
  └── has many → Favourite
  └── has many → Submission (as submitter)

Category (1) ─── has many ─── (M) Phrase
  └── belongs to exactly one category

Phrase (1) ─── has many ─── (M) Translation
  └── each translation is for one language
  └── has many → Favourite (via junction table)
  └── has many → Submission

Translation
  └── belongs to one phrase
  └── has status (Pending/Approved/Rejected)

Favourite (junction table between User and Phrase)
  └── belongs to one user
  └── belongs to one phrase

Submission (junction between User, Phrase, and Translation)
  └── belongs to one user (submitter)
  └── belongs to one phrase
  └── belongs to one language
  └── has status (Pending/Approved/Rejected)

LanguageStatistic
  └── logs each search and view
  └── stores language selected
  └── stores category
  └── stores timestamp and user (if authenticated)
```

**Preliminary Table Structure:**

```sql
-- ASP.NET Identity Tables (autogenerated)
-- Users, Roles, UserRoles, etc.

-- Custom Tables
Category
  - CategoryId (int, PK, identity)
  - Name (nvarchar(100), NOT NULL, UNIQUE)
  - Description (nvarchar(500), NULL)
  - IsActive (bit, NOT NULL, default: 1)
  - CreatedDate (datetime2, NOT NULL, default: GETDATE())
  - ModifiedDate (datetime2, NULL)

Phrase
  - PhraseId (int, PK, identity)
  - EnglishText (nvarchar(1000), NOT NULL)
  - CategoryId (int, FK → Category.CategoryId, NOT NULL)
  - IsActive (bit, NOT NULL, default: 1)
  - CreatedDate (datetime2, NOT NULL, default: GETDATE())
  - ModifiedDate (datetime2, NULL)

Translation
  - TranslationId (int, PK, identity)
  - PhraseId (int, FK → Phrase.PhraseId, NOT NULL)
  - Language (nvarchar(50), NOT NULL) -- "isiZulu", "isiXhosa", etc.
  - Text (nvarchar(2000), NOT NULL)
  - Status (nvarchar(20), NOT NULL) -- "Pending", "Approved", "Rejected"
  - RejectionReason (nvarchar(500), NULL)
  - SubmittedBy (nvarchar(450), FK → AspNetUsers.Id, NOT NULL)
  - ReviewedBy (nvarchar(450), FK → AspNetUsers.Id, NULL)
  - SubmittedDate (datetime2, NOT NULL, default: GETDATE())
  - ReviewedDate (datetime2, NULL)

Favourite
  - FavouriteId (int, PK, identity)
  - UserId (nvarchar(450), FK → AspNetUsers.Id, NOT NULL)
  - PhraseId (int, FK → Phrase.PhraseId, NOT NULL)
  - AddedDate (datetime2, NOT NULL, default: GETDATE())
  -- UNIQUE constraint on (UserId, PhraseId)

Submission (Community Translation Suggestion)
  - SubmissionId (int, PK, identity)
  - UserId (nvarchar(450), FK → AspNetUsers.Id, NOT NULL)
  - PhraseId (int, FK → Phrase.PhraseId, NOT NULL)
  - Language (nvarchar(50), NOT NULL)
  - SuggestedText (nvarchar(2000), NOT NULL)
  - Status (nvarchar(20), NOT NULL) -- "Pending", "Approved", "Rejected", "CorrectionRequested"
  - RejectionReason (nvarchar(500), NULL)
  - CorrectionInstructions (nvarchar(500), NULL)
  - SubmittedDate (datetime2, NOT NULL, default: GETDATE())
  - ReviewedDate (datetime2, NULL)
  -- UNIQUE constraint on (PhraseId, Language) for pending/approved to prevent duplicates

LanguageStatistic
  - StatisticId (int, PK, identity)
  - UserId (nvarchar(450), FK → AspNetUsers.Id, NULL) -- null if anonymous
  - PhraseId (int, FK → Phrase.PhraseId, NULL) -- null for search-only events
  - Language (nvarchar(50), NULL) -- null if no language filter
  - CategoryId (int, FK → Category.CategoryId, NULL) -- null if no category filter
  - EventType (nvarchar(20), NOT NULL) -- "View", "Search"
  - EventDate (datetime2, NOT NULL, default: GETDATE())
```

### Issue 4: Document Indexing Strategy

| Field | Value |
|-------|-------|
| **Title** | Document Database Indexing Strategy |
| **Labels** | `database`, `documentation`, `enhancement` |
| **Milestone** | Phase 2 – System Design |
| **Priority** | Medium |
| **Description** | Identify and document all database indexes needed for optimal performance, especially for search functionality. |

**Acceptance Criteria:**
- [ ] Indexes identified for frequently searched columns
- [ ] Indexes identified for foreign key columns
- [ ] Full-text search considered for phrase/translation search
- [ ] Index strategy documented in `docs/database-indexes.md`

**Preliminary Index Candidates:**

```sql
-- For Search Performance
CREATE INDEX IX_Phrase_EnglishText ON Phrase(EnglishText);
CREATE INDEX IX_Phrase_CategoryId ON Phrase(CategoryId);
CREATE INDEX IX_Translation_Language ON Translation(Language);
CREATE INDEX IX_Translation_Text ON Translation(Text); -- for search
CREATE INDEX IX_Translation_Status ON Translation(Status);
CREATE INDEX IX_Translation_PhraseId ON Translation(PhraseId);

-- For Favourites Performance
CREATE INDEX IX_Favourite_UserId ON Favourite(UserId);
CREATE UNIQUE INDEX IX_Favourite_Unique ON Favourite(UserId, PhraseId);

-- For Submissions Performance
CREATE INDEX IX_Submission_UserId ON Submission(UserId);
CREATE INDEX IX_Submission_Status ON Submission(Status);
CREATE INDEX IX_Submission_PhraseId ON Submission(PhraseId);
CREATE INDEX IX_Submission_Language ON Submission(Language);

-- For Statistics
CREATE INDEX IX_LanguageStatistic_User ON LanguageStatistic(UserId);
CREATE INDEX IX_LanguageStatistic_EventDate ON LanguageStatistic(EventDate);
CREATE INDEX IX_LanguageStatistic_Language ON LanguageStatistic(Language);
CREATE INDEX IX_LanguageStatistic_EventType ON LanguageStatistic(EventType);

-- For Category Management
CREATE INDEX IX_Category_IsActive ON Category(IsActive);
```

**Full-Text Search Consideration:**
For better search performance, consider adding a full-text index on `Phrase.EnglishText` and `Translation.Text` for more advanced search capabilities.
